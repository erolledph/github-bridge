Privacy Policy

Last updated: January 2024

Our Privacy Commitment

We don't access your account information - Your GitHub credentials are used only for authentication

We don't collect personal data - No personal information is stored on our servers

We don't store or steal user data - All data remains in your browser and GitHub account

How GitHub Bridge Works

GitHub Bridge is a client-side application that helps you upload your Bolt.new projects directly to GitHub repositories. Here's how we protect your privacy:

Authentication

  • You can authenticate using GitHub OAuth or by providing a personal access token
  • OAuth tokens are managed by Firebase Authentication and GitHub
  • Personal access tokens are encrypted and stored only in your browser's local storage
  • No authentication data is transmitted to or stored on our servers

Data Processing

  • All file processing happens in your browser
  • Files are uploaded directly from your browser to GitHub
  • We never see, store, or have access to your project files
  • Repository operations are performed directly with GitHub's API

Analytics

  • We use Google Analytics to understand how the application is used
  • Only anonymous usage data is collected (page views, button clicks)
  • No personal information or file content is included in analytics
  • You can disable analytics by blocking Google Analytics in your browser

Your Rights

Since we don't collect or store personal data:

  • There's no personal data to delete or modify
  • You can clear your browser's local storage to remove any stored tokens
  • You can revoke GitHub OAuth access through your GitHub settings
  • You can stop using the service at any time without any data concerns

Contact

If you have any questions about this privacy policy or our practices, please contact us through GitHub issues on our repository.
